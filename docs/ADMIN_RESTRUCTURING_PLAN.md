# 📋 خطة إعادة هيكلة لوحة الإدارة — عقارات السويداء
**التاريخ:** 2026-06-14  
**الحالة:** مقترحة  
**المرجع:** `docs/LOGIC_SPEC.md` + `DEVELOPMENT_GUIDELINES.md`

---

## 🎯 الرؤية

تحويل قسم الإدارة من "شبكة معقدة فيها 15+ بطاقة" إلى **نظام نظيف ومنظم حسب الأدوار**، بحيث:

- كل دور يرى **شاشته الخاصة فقط** (Role-Specific Dashboards).
- شاشة الإدارة الرئيسية تصبح **مركز إدارة الموظفين** (مستوحى من مشروع Final).
- كل العمليات الحساسة تتم عبر **Edge Functions** للأمان.
- الالتزام الكامل بـ **ميثاق المنطق** (`LOGIC_SPEC.md`).

---

## ❌ المشاكل الحالية

1. **داشبورد واحد كبير** (`/admin/dashboard`) يحتوي على كل شيء.
2. الصلاحيات تُستخدم فقط لإخفاء/إظهار البطاقات (يسبب تعقيد بصري).
3. لا يوجد شاشة مركزية لإدارة الموظفين (إضافة، تغيير دور، تعطيل، إعادة تعيين باسورد).
4. الموظف العادي (role=4) يرى أشياء كثيرة لا تخصه.
5. لا يوجد استخدام لـ Edge Functions في العمليات الإدارية الحساسة.

---

## ✅ الحل المقترح (الأنظف والأفضل)

### 1. هيكل الداشبوردات الجديد (Role-Specific)

| الدور              | الشاشة الجديدة              | المحتوى الرئيسي                          | الوصول |
|-------------------|-----------------------------|------------------------------------------|--------|
| **مدير (6)**      | `/admin/dashboard`          | إدارة الموظفين + إحصائيات + كل الأقسام  | كامل  |
| **نائب مدير (5)** | `/deputy/dashboard`         | إدارة الموظفين + معظم الأقسام           | محدود |
| **موظف مكتب (4)** | `/employee/dashboard`       | عروض + مواعيد + طلبات إتمام + مستخدمين  | محدود |
| **مشرف/منفذ (3)** | `/executor/tasks`           | مهام التنفيذ فقط                        | محدود |
| **مصور (2)**      | `/photographer/tasks`       | مهام التصوير فقط                        | محدود |

### 2. شاشة إدارة الموظفين (الأولوية الأولى)

ستصبح **`/admin/dashboard`** شاشة **إدارة الموظفين** فقط (مثل `admin_screen.dart` في مشروع Final)، وستحتوي على:

- قائمة الموظفين مع بحث
- إضافة موظف جديد (مع توليد باسورد تلقائي)
- تغيير الدور
- تعطيل/تفعيل الحساب
- إعادة تعيين كلمة السر
- حذف الموظف (ما عدا المدير الرئيسي)

### 3. استخدام Edge Functions (مطلوب)

جميع العمليات الحساسة ستتم عبر Edge Functions:
- `create-user`
- `update-user-role`
- `toggle-user-status`
- `reset-user-password`
- `delete-user`

## 🔒 الالتزام بالأمان (من LOGIC_SPEC.md)

- كل العمليات الحساسة ستستخدم **Edge Functions** فقط.
- لن يتم تعديل `role` أو `vrf` أو `sts` مباشرة من الـ Client.
- سيتم التحقق من صلاحية المدير قبل كل عملية.
- لا يُسمح بحذف المدير الرئيسي (`role=6`).
