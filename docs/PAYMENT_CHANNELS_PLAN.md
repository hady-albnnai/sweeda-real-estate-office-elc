## 🔄 الـ Flow الكامل

### 1️⃣ المستخدم — اختيار باقة:
```
packages_screen
  → اختار باقة (فضي/ذهبي)
  → ضغط "اشترك"
  → ينقل لـ payment_screen?pkg=X&amt=Y
```

### 2️⃣ المستخدم — اختيار قناة دفع:
```
payment_screen
  ↓
  يقرأ config.payChannels
  ↓
  يعرض فقط القنوات enabled
  ↓
  المستخدم يختار قناة (مثلاً "الهرم")
  ↓
  يظهر:
    - أيقونة + اسم القناة
    - بيانات المستقبل (اسم/هاتف)
    - التعليمات
    - حقل رقم العملية (ref)
    - حقل رفع صورة الإيصال (proof)
  ↓
  ضغط "تأكيد الدفع"
  ↓
  payments table INSERT (sts=0 قيد المراجعة)
  ↓
  notification داخلي للإدارة (لاحقاً)
```

### 3️⃣ الإدارة — التحقق والتفعيل:
```
admin/payments_screen (موجودة!)
  ↓
  قائمة الدفعات المعلّقة (sts=0)
  ↓
  لكل دفعة:
    - اسم المستخدم + الباقة
    - القناة + رقم العملية
    - صورة الإيصال (تكبير عند الضغط)
  ↓
  الإدارة تتحقق يدوياً:
    - الهرم: تتصل بفرع الهرم
    - شام كاش: تفتح تطبيق شام كاش وتشيك
    - رصيد: تشيك رصيد المستقبل
    - بنك: تشيك statement
  ↓
  ضغط "تفعيل" أو "رفض"
  ↓
  approvePayment() / rejectPayment()
  ↓
  trigger trg_payment_notify (E2+) → إشعار FCM تلقائي للمستخدم ✅
```

## 📋 خطوات التنفيذ (5 خطوات)

### الخطوة 1: Migration لإضافة `payChannels` لـ `app_config`
- ملف: `supabase/migrations/2026_06_06_payment_channels.sql`

- يقرأ `config.payChannels` بدل القنوات hardcoded

- زر حفظ مركّب

## 🧪 خطة الاختبار النهائي

### من المستخدم:
1. اختار باقة الفضية
2. اختار "شام كاش"
3. شاف الباركود + رقم الحساب
4. رفع صورة إيصال + كتب رقم عملية
5. ضغط "تأكيد"

### من الإدارة:
6. فتح `admin/payments` → شاف الدفعة الجديدة
7. ضغط على صورة الإيصال → شافها مكبّرة
8. ضغط "تفعيل"

### النتيجة:
9. ✅ المستخدم يستلم إشعار push: "✅ تم تفعيل اشتراكك"
10. ✅ المستخدم يفتح Profile → يشوف الباقة الفضية مفعّلة
11. ✅ المستخدم يقدر يضيف عروض حسب حد الباقة الجديد (15 عرض بدل 5)
