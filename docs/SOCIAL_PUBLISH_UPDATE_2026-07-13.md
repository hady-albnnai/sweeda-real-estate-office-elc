# تحديث النشر التلقائي على السوشيال + صفحات التواصل (2026-07-13)

## التنفيذ المكتمل

### 1. صفحات التواصل الاجتماعي (قابلة للتعديل من الإدارة)
- **مفاتيح جديدة في `app_config.txts`**:
  - `facebook`: رابط صفحة فيسبوك
  - `instagram`: رابط حساب إنستغرام
  - `developerPhone`: رقم هاتف المطور (يظهر في "عن التطبيق")
  - `socialPages`: خريطة إضافية لصفحات جديدة (تيك توك، إلخ)

- **عرض في "عن التطبيق"** (`settings_screen.dart`):
  - روابط قابلة للضغط (فيسبوك + إنستغرام + صفحات إضافية)
  - قسم المطور: لوجو `loraneem_tech_logo.png` + الاسم + التاجلاين + رقم الهاتف (من الكونفيغ)

- **إدارة في `/admin/config`**:
  - حقول لـ فيسبوك + إنستغرام + رقم المطور
  - إضافة/حذف صفحات إضافية ديناميكياً

### 2. منطق النشر التلقائي (كما طلب المستخدم)
- **في شاشة إضافة عرض** (`add_offer_screen.dart`):
  - تشيك بوكس مفعّل **تلقائياً** (يمكن إلغاؤه)
  - النص: "نشر العرض تلقائياً على صفحاتنا في فيسبوك وإنستغرام (وأي صفحات إضافية مضافة)"
  - عند الضغط على "نشر العرض للمراجعة الآن":
    - `i_soc = 1` (إذا مفعّل)
    - `socTxt` = **قالب جاهز** يُولّد تلقائياً من بيانات العرض

- **قالب المنشور** (مولّد داخل التطبيق):
  ```
  🏠 عرض جديد على عقارات السويداء

  📍 [العنوان]
  [عقار/سيارة] [للبيع/للإيجار]
  💰 السعر: ...
  📌 المنطقة: ...
  [الوصف]

  للتفاصيل والحجز:
  📱 تطبيق عقارات السويداء
  #عقارات_السويداء #عروض_عقارية
  ```

- **التدفق**:
  - العرض يذهب للمراجعة (`sts=0`)
  - بعد الموافقة (`sts=2`): إذا `i_soc=1` يُستخدم `socTxt`
  - (النشر الفعلي عبر Meta API مؤجل — يُستخدم النص الجاهز يدوياً أو عبر Edge Function لاحقاً)

### 3. التحديثات على التوثيق
