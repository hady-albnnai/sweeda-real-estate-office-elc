# شاشة فحص النظام

تمت إضافة شاشة داخل التطبيق لتقليل الاختبار اليدوي قدر الإمكان.

## المسار

```txt
/admin/qa
```

وتظهر في لوحة الإدارة باسم:

```txt
فحص النظام
```

## ماذا تفحص؟

### محلياً داخل التطبيق

- وجود `userModel`.
- حالة تسجيل الدخول.
- هل الدور إداري.
- هل الصلاحيات مقروءة.

### من السيرفر عبر RPC

تستدعي:

```sql
qa_system_check(p_admin_uid)
```

وتفحص:

- الجداول الأساسية.
- الأعمدة المهمة.
- الدوال الجديدة والقديمة المهمة.
- RLS على الجداول المهمة.
- فهرس منع تكرار الهاتف.
- وجود أرقام هاتف مكررة بعد التطبيع.
- ملخص عدد المستخدمين والعروض ومهام التصوير.

## المطلوب قبل استخدامها

نفذ migration:

```txt
supabase/migrations/2026_06_10_qa_system_check.sql
```

## طريقة الاستخدام

1. ادخل بحساب مدير.
2. افتح لوحة الإدارة.
3. اضغط `فحص النظام`.
4. اضغط `تشغيل الفحص`.
5. أرسل النتائج إذا ظهر أي `FAIL`.

## ملاحظة

هذه الشاشة لا تغني عن الاختبار اليدوي بالكامل، لكنها تختصر معظم فحص السيرفر والصلاحيات والدوال.


## التوسعة القصوى 2026-06-10

تم توسيع `qa_system_check` ليشمل فحوصات إضافية:

- جداول أكثر، منها `ratings`.
- أعمدة تفصيلية للمستخدمين والعروض والمواعيد والمدفوعات والتصوير.
- دوال RPC أكثر، منها دوال OTP، الإحالة، FCM، الدفع، التوثيق، التصوير، والفحص نفسه.
- فحص محتوى بعض الدوال المهمة مثل تطبيع الهاتف داخل `upsert_user_after_otp` ووجود مفاتيح التصوير في دالة الصلاحيات.
- فحص RLS على جداول حساسة.
- فحص عدد السياسات على الجداول المهمة.
- فحص الفهارس الأساسية.
- فحص Storage buckets.
- فحص `app_config/main` ومفاتيح Config الأساسية.
- فحص سلامة البيانات:
  - مستخدمين بلا وسيلة تواصل.
  - عروض بلا مالك.
  - عروض منشورة بلا صور.
  - عروض منشورة بلا موقع.
  - مواعيد مرتبطة بعرض محذوف/غير موجود.
  - مهام تصوير بلا عرض.
  - مهام تصوير بلا مصور.
  - مهام مرسلة بلا وسائط.
- إظهار عدادات الطوابير: مدفوعات معلقة، تبليغات مفتوحة، توثيق بانتظار المراجعة.
