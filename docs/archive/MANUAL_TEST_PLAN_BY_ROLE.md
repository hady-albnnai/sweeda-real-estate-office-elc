# دليل الاختبار اليدوي الشامل حسب الدور

**المشروع:** المكتب العقاري الإلكتروني — عقارات السويداء  
**آخر تحديث:** 2026-06-17  
**الغرض:** توزيع هذا الملف على فريق الاختبار بحيث يختبر كل شخص التطبيق حسب الدور المخصص له، مع توثيق النتيجة والملاحظات بدقة.

---

## 1. طريقة استخدام الدليل

لكل اختبار يوجد:

- **رقم الاختبار:** مثال `VIS-01`.
- **الدور:** الزائر، المستخدم، الوسيط، المصور، المشرف، موظف المكتب، نائب المدير، المدير.
- **الخطوات:** ماذا يفعل المختبر بالضبط.
- **النتيجة المتوقعة:** ماذا يجب أن يحصل إذا كان التطبيق صحيحاً.
- **النتيجة الفعلية:** يكتبها المختبر.
- **الحالة:** `نجح / فشل / يحتاج مراجعة`.
- **ملاحظات:** لقطة شاشة، رسالة خطأ، أو وصف المشكلة.

> يفضّل أن يتم الاختبار على جهازين على الأقل: Android و iOS إن أمكن، وعلى شبكة مختلفة مرة واحدة على الأقل.

## 2. قواعد عامة قبل البدء

### 2.1 بيئة الاختبار

| البند | القيمة |
|---|---|
| نسخة التطبيق | اكتب رقم النسخة أو آخر commit |
| تاريخ الاختبار | اكتب التاريخ |
| نوع الجهاز | Android / iOS / Web إن وجد |
| الحساب المستخدم | اكتب الاسم ورقم الهاتف والدور |
| الشبكة | Wi-Fi / 4G / 5G |

### 2.2 ملاحظات مهمة

- لا تستخدم بيانات حقيقية حساسة أثناء الاختبار إلا إذا كانت بيئة الإنتاج وتحت صلاحية المدير.
- عند اختبار صور الهوية، استخدم صور اختبار واضحة، ويفضل صورتين: وجه وقفا.
- عند اختبار الدفع، استخدم إثبات دفع تجريبي واضح عليه كلمة `TEST` إن أمكن.
- لا تحذف بيانات حقيقية إلا إذا كانت مخصصة للاختبار.
- عند ظهور خطأ، يجب توثيق:
  - اسم الشاشة.
  - الدور المستخدم.
  - الخطوات التي سبقت الخطأ.
  - نص الخطأ كما هو.
  - لقطة شاشة إن أمكن.

## 3. مصفوفة الأدوار والصلاحيات الأساسية

| الدور | رقم الدور | الغرض الرئيسي | ملاحظات |
|---|---:|---|---|
| زائر | بدون تسجيل دخول | تصفح وبحث وعرض تفاصيل | لا يستطيع الحجز أو التبليغ إلا بعد تسجيل الدخول حسب الشاشة |
| مستخدم عادي | 0 | إضافة عروض، طلبات، حجز مواعيد، مفضلة، دفع، توثيق | هو الدور الأساسي للزبون |
| وسيط | 1 / brk=1 | لوحة وسيط وعروض ومواعيد وصفقات وإحصائيات | الوساطة تعتمد على `brk` وليس فقط role |
| مصور | 2 | مهام التصوير | موظف داخلي محدود |
| مشرف | 3 | متابعة ميدانية/إدارية حسب الصلاحيات | يصل لأجزاء إدارية حسب الصلاحيات |
| موظف مكتب | 4 | عمليات المكتب اليومية | مراجعة عروض، طلبات، مواعيد... حسب الصلاحيات |
| نائب مدير | 5 | إدارة عليا جزئية | لا يدير نائب مدير آخر ولا المدير |
| مدير | 6 | كل الصلاحيات | يستطيع إدارة الموظفين والصلاحيات والإعدادات |

## 19. ملاحظات تنفيذية لفريق الإدارة

- بعد أي تحديث يحتوي Edge Functions، يجب نشر الدوال المطلوبة قبل الاختبار.
- بعد تحديث إدارة الموظفين الحالي، يجب التأكد من نشر:
  - `create-user`
  - `get-staff-id-images`
- إذا كان الموظف مضافاً قديماً بصورة واحدة، يجب أن يظهر زر عرض الصور بعدد 1.
- إذا كان `users.img` فارغاً، سيظهر أنه لا توجد صور هوية محفوظة.
- إذا احتاج المختبر فحص ذلك من SQL:

```sql
SELECT id, nm, ph, role, sid, ad, img, ts_crt
FROM public.users
WHERE i_del = 0 AND role >= 2
ORDER BY role DESC, ts_crt DESC;
```
