## الوضع الحالي

كل المسارات الحرجة أصبحت تعتمد على RPCs، وتم إصلاح كل المشاكل المكتشفة.
**التطبيق جاهز للاختبار الفعلي.**

---

## 1. الاختبار الفعلي (الأولوية القصوى)

اتبع خطة الاختبار الكاملة في:

```txt
docs/POST_FIX_EXECUTION_AND_TEST_PLAN.md
```

الخطوات:
```bash
git pull origin main
flutter pub get
flutter analyze
flutter run
```

ثم اختبر بالترتيب:
1. العروض (إضافة، مراجعة، قبول/رفض)
2. المواعيد (حجز، تراشق، موافقة/رفض)
3. الطلبات
4. المدفوعات
5. التقييمات
6. نظام المصور

---

## 2. تفعيل WhatsApp OTP الإنتاجي

**الحالة:** dev fallback نشط. Edge Functions موجودة لكن تحتاج Meta credentials.

**المرجع:**
```txt
docs/WHATSAPP_ACTIVATION_PLAN.md
docs/AUTH_SETUP.md
```

**المطلوب:**
- إعداد Meta Business + WhatsApp Cloud API
- ضبط secrets على Supabase
- نشر Edge Functions
- اختبار `auth.uid()` في الجلسات الإنتاجية

---

## 3. تشديد `auth.uid()` الكامل (بعد الإنتاج)

**الحالة:** RPCs محمية جزئياً — `auth.uid()` يُفحص إذا كان متاحاً، وإلا يُقبل الـ UID من العميل.

**عند تفعيل Auth إنتاجي:** نعود ونشدد كل RPCs لترفض أي طلب بدون `auth.uid()`.

**المرجع:**
```txt
docs/SECURITY_REVIEW.md
```

---

## 4. اختبار نظام المصور

**الحالة:** منفذ برمجياً، يحتاج اختبار عملي.

**الخطوات:**
- منح مستخدم صلاحية `photographer_tasks`
- إنشاء مهمة من `/admin/photography-management`
- دخول المصور إلى `/photographer/tasks`
- رفع الصور وإرسالها
- اعتماد وربط الصور بالعرض

---

## 5. تعميم التطبيق لكل سوريا (قرار تجاري)

**الحالة:** الاسم عام "المكتب العقاري الالكتروني"، لكن البيانات/Config مرتبطة بالسويداء.

**المطلوب لاحقاً:**
- تحديث `docs/locations.json` أو Config ليشمل المحافظات السورية
- مراجعة النصوص المتبقية
