# تدقيق الميزات الحالي

> آخر تحديث: 2026-06-10

هذا الملف كان سابقاً يحتوي تاريخاً طويلاً من مراحل منفذة وملاحظات قديمة. تم اختصاره ليعكس الحالة الحالية فقط، حتى لا تربك المحادثات القادمة.

## الميزات المكتملة حالياً

### النواة الأساسية

- Flutter + Provider + GoRouter.
- Supabase PostgreSQL + Storage + Realtime + RPC.
- Auth عبر WhatsApp OTP / Email Magic Link مع dev fallback.
- Config-driven business settings.
- نماذج بيانات بأسماء قصيرة متوافقة مع `docs/SPEC.md`.

### واجهات المستخدم

- واجهة زائر.
- واجهة مستخدم.
- إضافة/تعديل/عرض عروض.
- الطلبات.
- المواعيد.
- المفضلة.
- الملف الشخصي.
- الباقات والمدفوعات.
- التقييمات.

### الوسيط

- لوحة الوسيط.

- إحصائيات الوسيط.

### الإدارة

- لوحة الإدارة.
- إدارة المستخدمين.

### الإدارة الداخلية الجديدة

- حماية المسارات حسب الدور والصلاحيات.
- صلاحيات ديناميكية عبر `users.perm`.
- شاشة إدارة الصلاحيات `/admin/permissions`.
- مركز عمليات المكتب `/admin/office-operations`.
- إدارة الوسائط `/admin/media-review`.
- إدارة مهام التصوير `/admin/photography-management`.
- شاشة مهام المصور `/photographer/tasks`.

### إصلاحات مهمة

- تغيير دور المستخدم وحالته عبر RPC بدل تحديث مباشر.
- منع تكرار رقم الهاتف بصيغ مختلفة.
- إعفاء الإدارة من حدود باقات المستخدم عند إضافة عروض.
- إنشاء العرض عبر RPC للتوافق مع وضع التطوير الحالي.
- إزالة `print` المتبقية من الكود.
- تحديث اسم التطبيق المثبت إلى `المكتب العقاري الالكتروني`.
- تصحيح مقاسات الأيقونات.
- تحسين تنقل الأدوار من أعلى الصفحة.
- إصلاح overflow في كروت لوحة الإدارة.

## 🛠️ تحديث 2026-06-10 — تطبيع الهاتف في upsert_user_after_otp

تم تحديث دالة `upsert_user_after_otp` لتستخدم `normalize_sy_phone` في مسار WhatsApp/SMS، لأن وضع التطوير يعتمد عليها عند تسجيل الدخول. هذا يكمل منع تكرار الحسابات لنفس رقم الهاتف.

## 🎨 تحديث 2026-06-10 — نص حصة العروض للحساب الإداري

تم تعديل شريط الحصة في شاشة إضافة العرض ليعرض للحسابات الداخلية `role >= 2`:

`حساب إداري — إضافة العروض غير محدودة`

بدلاً من إظهار حد رقمي كبير مع عبارة باقة مجانية.

## 🛠️ تحديث 2026-06-10 — RPCs مهام التصوير لوضع التطوير

تم تحويل إنشاء/إرسال/اعتماد مهام التصوير إلى RPCs متوافقة مع وضع المصادقة التطويري، بدل الاعتماد على INSERT/UPDATE مباشر قد يفشل بسبب RLS عندما لا تكون `auth.uid()` متاحة.
