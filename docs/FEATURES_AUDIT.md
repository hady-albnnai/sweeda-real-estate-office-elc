## الميزات المكتملة حالياً

### النواة الأساسية

- Flutter + Provider + GoRouter.
- Supabase PostgreSQL + Storage + Realtime + RPC.
- Auth عبر WhatsApp OTP / Email Magic Link مع dev fallback.
- Config-driven business settings.
- نماذج بيانات بأسماء قصيرة متوافقة مع `docs/SPEC.md`.

### إصلاحات مهمة

- تغيير دور المستخدم وحالته عبر RPC بدل تحديث مباشر.
- منع تكرار رقم الهاتف بصيغ مختلفة.
- إعفاء الإدارة من حدود باقات المستخدم عند إضافة عروض.
- إنشاء العرض عبر RPC للتوافق مع وضع التطوير الحالي.
- تشديد منطق إنشاء العرض: حصة + كشف التكرار + الإرجاع إلى حالة المراجعة.
- توحيد حالات المواعيد بين المستخدم/الوسيط/الإدارة وإضافة `appointments.req_uid` لطالب الموعد.
- نقل أسعار الباقات وسعر الصرف إلى Config (`pkg.*.pr` + `fx.usd_syp`).
- تحويل قوائم المناطق في إضافة العرض إلى القراءة من Config بدل fallbackات محلية ثقيلة.
- إصلاح شاشة "عروضي" لتجلب كل عروض المستخدم لا المنشور العام فقط.
- إلغاء التحديث اليدوي المزدوج لبعض الإحصائيات والاعتماد على triggers.
- تحسين خصوصية بيانات المالكين باستخدام `users_public` في القراءة العامة.
- تجهيز دفعة RPCs إضافية لتثبيت ما قبل الاختبار الحقيقي (request/payment/report/appointment/admin review/notification/rating/profile...).
- إزالة `print` المتبقية من الكود.
- تحديث اسم التطبيق المثبت إلى `المكتب العقاري الالكتروني`.
- تصحيح مقاسات الأيقونات.
- تحسين تنقل الأدوار من أعلى الصفحة.
- إصلاح overflow في كروت لوحة الإدارة.

### نظام المنفذ الميداني (2026-06-12)

- شاشة مهام المنفذ `/executor/tasks` — 4 تبويبات (اليوم / المؤجلة / المنفذة / بانتظار المراجعة).
- شاشة تنفيذ المهمة `/executor/execute/:id` — 3 إجراءات (تأجيل / رفض / طلب إتمام).
- جدول `completion_requests` لطلبات الإتمام.
- 7 دوال RPC جديدة (منفذ + مكتب).

- المكتب (مدير/نائب/موظف) يراجع طلبات الإتمام ويوافق/يرفض.
