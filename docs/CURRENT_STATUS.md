# الحالة الحالية للمشروع — المكتب العقاري الالكتروني

> آخر تحديث: 2026-06-10
> المستودع الصحيح: `hady-albnnai/sweeda-real-estate-office-elc`

## ملخص تنفيذي

المشروع حالياً أصبح تطبيق مكتب عقاري إلكتروني مبني على Flutter + Supabase، مع:

- واجهة زائر ومستخدم.
- لوحة وسيط.
- لوحة إدارة.
- نظام صلاحيات داخلي ديناميكي.
- مركز عمليات المكتب.
- إدارة مهام التصوير والمصور.
- إدارة الوسائط.
- حماية مسارات حسب الدور والصلاحية.
- دعم وضع التطوير الحالي لتسجيل WhatsApp OTP fallback.

## اسم التطبيق المثبت

```txt
المكتب العقاري الالكتروني
```

مع استمرار وجود هوية/منطقة عقارات السويداء داخل المحتوى الحالي، ليبقى التطبيق قابلاً للتوسع لاحقاً لكل سوريا.

## الأدوار الحالية

يعتمد المشروع على `users.role`:

| role | الدور |
|---:|---|
| 0 | مستخدم |
| 1 | وسيط |
| 2 | مشرف / موظف تشغيل |
| 3 | نائب مدير |
| 4 | مدير |

## الصلاحيات الديناميكية

تمت إضافة الحقل:

```sql
users.perm JSONB DEFAULT '[]'
```

القاعدة:

- `perm = []` → يستخدم التطبيق صلاحيات الدور الافتراضية.
- `perm` غير فارغة → تستخدم كصلاحيات مخصصة.

أهم الشاشات:

| الشاشة | المسار | الصلاحية |
|---|---|---|
| لوحة الإدارة | `/admin/dashboard` | role >= 2 |
| عمليات المكتب | `/admin/office-operations` | `office_operations` |
| إدارة الصلاحيات | `/admin/permissions` | `manage_permissions` |
| إدارة الوسائط | `/admin/media-review` | `media_review` |
| إدارة مهام التصوير | `/admin/photography-management` | `photography_management` |
| مهام المصور | `/photographer/tasks` | `photographer_tasks` |

## الإدارة الداخلية

تم تنفيذ:

- حماية مسارات الإدارة والوسيط والمستخدم والمصور.
- مركز عمليات المكتب.
- شاشة إدارة الصلاحيات.
- شاشة إدارة الوسائط.
- شاشة إدارة مهام التصوير.
- شاشة مهام المصور.
- إصلاح تغيير دور المستخدم وحالته عبر RPC.
- منع تكرار رقم الهاتف بصيغ مختلفة.
- إعفاء حسابات الإدارة `role >= 2` من قيود حصة العروض.
- إنشاء العرض عبر RPC لتفادي مشاكل RLS في وضع التطوير.

## وضع المصادقة الحالي

يوجد WhatsApp OTP مع Edge Functions، لكن يوجد أيضاً dev fallback.

هذا يعني أن بعض الدوال القديمة التي تعتمد على `auth.uid()` قد تفشل في وضع التطوير. لذلك أُضيفت دوال إدارية متوافقة مع الوضع الحالي وتقبل `p_admin_uid` مع فحص دوره من جدول `users`.
