## ✅ النقطة 1: الزائر (Visitor) — معتمدة

**الهوية:** أي شخص حمّل التطبيق وفتحه بدون تسجيل. يدخل مباشرة كزائر في وضع "مشاهدة فقط" (read-only).

### يقدر:
- تصفّح تبويب **العروض** كاملاً (عقارات/سيارات → بيع/إيجار).
- تصفّح تبويب **الطلبات** كاملاً (عقارات/سيارات → شراء/استئجار).
- فتح **تفاصيل أي عرض** (صور، سعر، مواصفات، وصف، موقع) — **عدا بيانات صاحب العرض**.
- **البحث والفلترة** بحرية (شرط ألا تضيف تكلفة سيرفر كبيرة → يفضّل فلترة محلية/مُحسّنة).
- **مشاركة العرض** — متاحة للزائر **بدون تسجيل دخول** (أداة تسويق).

### لا يقدر (أي تفاعل):
- حجز موعد، التواصل مع الإدارة للحجز، رفع عرض، إضافة طلب، حفظ مفضلة.
- أي إجراء تفاعلي → نافذة منبثقة: **"أنت بحاجة لتسجيل الدخول للقيام بهذا الإجراء"** + زر تسجيل دخول داخل نفس النافذة.

### قرارات مهمة مثبّتة:
1. **بيانات صاحب العرض (الاسم/الهاتف) مخفية عن الجميع** (زائر + مستخدم + سمسار)، وحصرية للإدارة فقط. → هذه **نواة نموذج الربح** (تفاصيله تُناقَش لاحقاً).
2. **لا يوجد زر "اتصال مباشر بصاحب العرض"** إطلاقاً. آلية التواصل حول أي عرض = خياران فقط:
   - **(أ) حجز موعد من داخل التطبيق** (يمر عبر الإدارة أو السمسار حسب مصدر العرض).
   - **(ب) التواصل مع الإدارة (المكتب) لحجز موعد** عن طريقهم.
   - الخياران يظهران للزائر، لكن الضغط عليهما → نافذة "سجّل دخولك".
3. **المشاركة = Deep Link**:
   - رابط يفتح **هذا العرض تحديداً** داخل التطبيق إن كان مثبّتاً.
   - إن لم يكن مثبّتاً → تحويل إلى **Google Play** للتحميل، ثم فتح العرض (Deferred Deep Link).
4. **المحتوى المعروض للزائر = نفس محتوى المستخدم المسجّل تماماً**. الفرق الوحيد = القدرة على التفاعل.
