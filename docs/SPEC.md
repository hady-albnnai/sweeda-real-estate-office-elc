## ✅ النقطة 1: الزائر (Visitor)

**الهوية:** أي شخص حمّل التطبيق وفتحه بدون تسجيل. يدخل مباشرة كزائر في وضع "مشاهدة فقط" (read-only).

### يقدر:
- تصفّح تبويب **العروض** كاملاً (عقارات/سيارات → بيع/إيجار).
- فتح **تفاصيل أي عرض** (صور، سعر، مواصفات، وصف، موقع) — **عدا بيانات صاحب العرض**.
- **البحث والفلترة** بحرية (فلترة محلية/مُحسّنة لتجنب تكلفة سيرفر).
- **مشاركة العرض** — متاحة للزائر **بدون تسجيل دخول** (أداة تسويق).

### لا يقدر (أي تفاعل):
- حجز موعد، التواصل مع الإدارة للحجز، رفع عرض، إضافة طلب، حفظ مفضلة.
- أي إجراء تفاعلي → نافذة منبثقة: **"أنت بحاجة لتسجيل الدخول للقيام بهذا الإجراء"** + زر تسجيل دخول داخل نفس النافذة.

### قرارات مهمة مثبّتة:
1. **بيانات صاحب العرض (الاسم/الهاتف) مخفية عن الجميع** (زائر + مستخدم + سمسار)، وحصرية للإدارة فقط. → نواة نموذج الربح.
2. **لا يوجد زر "اتصال مباشر بصاحب العرض"** إطلاقاً.
3. **المشاركة = Deep Link**: رابط يفتح العرض داخل التطبيق. إن لم يكن مثبّتاً → Google Play للتحميل.
4. **المحتوى المعروض للزائر = نفس محتوى المستخدم المسجّل**. الفرق الوحيد = القدرة على التفاعل.

## ✅ النقطة 2: المستخدم (User) والتفعيل

### التسجيل والتفعيل
- التسجيل/الدخول عبر **رقم الموبايل فقط + رمز تحقق OTP**.
- **الأولوية للأرقام السورية** (09 / +963)، لكن يُقبل أي رقم دولي.
- **كل رقم موبايل = مستخدم واحد**. لا يُسمح بأكثر من **جلسة واحدة** نشطة.
- **لا يمكن تعديل رقم الموبايل** بعد التفعيل.
- التفاصيل (الاسم الثلاثي، العنوان...) تُؤخذ **عند أول إجراء** (إضافة عرض/طلب/حجز موعد) لا عند التسجيل.

### حذف الحساب
- يستطيع المستخدم **حذف حسابه** ذاتياً (متطلب Google Play).
- عند الحذف: تُحذف **كل عروضه وطلباته وحجوزاته تلقائياً**، مع **إشعار الإدارة**.

### الخصوصية
- **كل العروض والطلبات تظهر للجمهور باسم "الإدارة" فقط**.

### الحدود (Quotas)

| البند | المستخدم العادي | الوسيط |
|---|---|---|
| عروض مجانية | 1 | 5 |
| طلبات | 3 / شهر | 5 / شهر |
| مواعيد مجانية | 3 / شهر | 3 / شهر |

- **عدّاد المواعيد يُحسب عند الإنشاء** (لا عند التأكيد).

### رمز التحقق OTP
- صلاحية الرمز: **دقيقتان (120 ثانية)** + زر "إعادة إرسال" يظهر بعد انتهاء العدّاد.

### المفضلة
- **غير محدودة**، وتُخزَّن **محلياً على جهاز المستخدم**.
- الإدارة تُشعَر بـ **عداد عند الضغطة فقط** (مثلاً: "العرض أضيف للمفضلة 47 مرة").

## ✅ النقطة 3: الهيكل العام للواجهة

### التبويبات السفلية (3 تبويبات فقط)
```
🏠 العروض        ➕ إضافة        👤 حسابي
```

### الشريط العلوي (ثابت)
- **اليسار:** 💬 **زر التواصل مع المكتب** → يفتح مجموعة واتساب مخصّصة.
- **الوسط:** 🏛️ شعار التطبيق.
- **اليمين:** 🔔 **جرس الإشعارات**.

### تبويب «العروض» (الرئيسي)
- تبويبان رئيسيان: **🏢 عقارات** / **🚗 سيارات**.
- داخل كل تبويب: **بيع** / **إيجار**.
- **فلتر الموقع أعلى كل تبويب** (قائمة كبيرة + حقل إدخال حر).
- بطاقات العروض مع **زر تفضيل 🤍** على كل بطاقة.
- ضغط البطاقة → **شاشة تفاصيل العرض**.
- **الترتيب:** الأحدث → الأقدم، **عدا الباقات** (ذهبي > فضي > عادي).
- **الفلاتر:** لوحة منزلقة من الأسفل (Bottom Sheet).

### شاشة التفاصيل (Details Screen)
1. **شريط الصور** (Carousel)
2. **قسم السعر** (بارز) — السعر الإجمالي + "شامل عمولة المكتب"
3. **بادج مصدر العرض**
4. **المواصفات الرئيسية**
5. **الموقع** — التقريبي فقط
6. **الوصف**
7. **معرض الصور**

**الأزرار الثابتة في الأسفل:**
- 🔴 **حجز موعد** (الزر الرئيسي)
- 📤 **مشاركة**
- 🤍 **إضافة للمفضلة**

### زر «الإضافة» (➕ الأوسط)
- **"أضيف عرض"** (عندي عقار/سيارة للبيع/الإيجار)
- **"أضيف طلب"** (أبحث عن عقار/سيارة — يُرسل للإدارة سراً)

### تبويب «حسابي»
```
👤 حسابي
   ├── 📦 عروضي
   ├── 📋 طلباتي
   ├── 📅 مواعيدي
   ├── ❤️ المفضلة (محلية)
   ├── 💎 باقتي/الاشتراك
   ├── 🏆 تقدمي (نقاط + بادج)
   └── ⚙️ إعدادات الحساب + تسجيل خروج
```

- **الزائر:** ضغط «حسابي» → شاشة **"سجّل دخولك"**.

### شاشة البداية (Splash)
- شاشة افتتاحية عصرية بشعار المكتب.

### قرارات عامة
- **عربي صرف** (RTL كامل، خط Cairo). لا لغة ثانية.
- **بلا وضع ليلي** (Dark Mode).

### 4-ج: استمارة المستخدم العادي (إضافة عرض)

#### حقول إضافية إلزامية:
- **المواعيد المتاحة للحجز:** الأيام + الساعات المتاحة في كل يوم.

#### التدفق العام:
1. التحقق من توفر **خانة عرض مجانية** (1 فقط للمستخدم العادي)
2. اختيار نوع العرض: عقار أو سيارة
3. ملء الحقول المعتمدة
4. إدخال **السعر الصافي** ← النظام يحسب ويعرض السعر الإجمالي
5. تظهر **عمولة المكتب** كمعلومة فقط
6. إدخال **الاسم الثلاثي + العنوان**
7. رفع حتى 6 صور + خيار فيديو عبر واتساب
8. ☑️ **checkbox مشاركة على السوشيال ميديا** (اختياري)
9. **📝 التعهد الإلكتروني** (إلزامي للمتابعة)
10. حفظ كمسودة محلياً
11. عرض **ملخص** + تنويه "سيُراجع من الإدارة"
12. الإرسال → **قيد المراجعة**

### 4-د: استمارة الوسيط/الشركة (إضافة عرض)

#### حقول إضافية:
- المواعيد المتاحة للحجز (إلزامي)
- **نسبة عمولة الوسيط** (حتى 5%) — تظهر للإدارة فقط
- **نوع الناشر:** وسيط فردي / شركة

#### التدفق العام:
1. التحقق من توفر الخانات (5 مجانية)
2. اختيار نوع العرض
3. ملء الحقول
4. إدخال السعر الصافي
5. تحديد نسبة عمولة الوسيط (حتى 5%)
6. إدخال بيانات صاحب العرض (الاسم + الهاتف) — سرية للإدارة
7. رفع الصور والفيديو
8. ☑️ checkbox السوشيال ميديا (اختياري)
9. 📝 التعهد الإلكتروني (إلزامي)
10. حفظ كمسودة
11. ملخص + إرسال → **قيد المراجعة**

### 4-هـ: ما يظهر للإدارة (شاشة مراجعة العرض)

**القسم الأول: معلومات العرض** — كل الحقول
**القسم الثاني: معلومات الناشر (سرية)** — الاسم + الهاتف + العنوان
**القسم الثالث: معلومات الوسيط (إن وجد)** — الاسم + نسبة العمولة
**القسم الرابع: العمولة** — عمولة المكتب + عمولة الوسيط (قابلة للتعديل)
**القسم الخامس: المرفقات السرية** — صورة سند الملكية + الموقع الدقيق
**القسم السادس: ملاحظات داخلية** — حقل نصي حر

**الإجراءات:** نشر / رفض مع سبب / طلب تعديل / أرشفة

## ✅ النقطة 6: رفع العرض والمراجعة

### دورة حياة العرض
```
مسوّدة → قيد المراجعة → (منشور | مرفوض مع سبب | معدّل)
```

### مدة العرض والتجديد
- **مدة العرض**: 30 يوم
- **إشعار ما قبل الانتهاء**: قبل 48 ساعة + قبل 24 ساعة
- **التجديد**: بضغطة واحدة دون إعادة إدخال البيانات
- **مدة السماح للتجديد**: شهر كامل بعد الانتهاء
- **إذا لم يُجدد**: يُحذف العرض بالكامل (تبقى الإحصائيات فقط)

### المراجعة
- كل العروض تمر بمراجعة الإدارة (بما فيها عروض الإدارة نفسها).

### العروض المكررة
- النظام يستخدم **بصمة رقمية** (صور، عنوان، موقع) لكشف التكرار المحتمل.
- عند اكتشاف تكرار → **تحذير للإدارة** في شاشة المراجعة.
- للإدارة صلاحية **تجاوز التحذير ونشر العرض** إذا النظام ما كان دقيقاً.
- عند الرفض كـ "مكرر" → يُسجّل السبب.

## ✅ النقطة 7: حجز المواعيد

### مبدأ عام
- كل الحجوزات تمر عبر الإدارة.
- صاحب العرض يحدد المواعيد المتاحة عند إضافة العرض.

### آلية حجز الموعد
1. المستخدم يضغط **"حجز موعد"** في تفاصيل العرض
2. تظهر **قائمة المواعيد المتاحة فقط** (مع مراعاة المحجوزة مسبقاً)
3. **الفرق بين أي موعدين ساعة كاملة** على الأقل
4. يختار الموعد ويرسل الطلب
5. يُرسل إشعار لـ (طالب الحجز + الإدارة + الوسيط إن وجد)

### دور الإدارة
- تراجع طلب الحجز → موافقة / اقتراح بديل / رفض مع سبب

### الإلغاء والتعديل
- يمكن طلب الإلغاء/التعديل **قبل 5 ساعات** على الأقل
- إذا أقرب → **الإلغاء فقط** (لا تعديل)

### شاشة تنبيه المواعيد السابقة
عند حجز موعد لِعرض عليه مواعيد سابقة، تظهر منبثقة:
```
⚠️ هذا العرض عليه (3) مواعيد سابقة
قد يتم بيعه قبل موعدك إذا ضغط أحدهم "إتمام المعاملة"
هل تريد حجز موعد مع العلم بذلك؟
[🔵 نعم، أحجز]  [🔴 إلغاء]
```

### أولوية الضاغط أولاً
- عندما يضغط أحد الأطراف **"إتمام المعاملة"** ← العرض يُحجز فوراً
- تُلغى تلقائياً باقي المواعيد المعلقة للعرض نفسه
- يُرسل إشعار لأصحاب المواعيد الأخرى: "نأسف، تم بيع/تأجير العقار"

## ✅ النقطة 8: موافقة الوسيط وصاحب العرض على المواعيد

### مبدأ عام
- الإدارة **لا توافق** على مواعيد المستخدمين/الوسطاء.
- الإدارة توافق فقط على مواعيد عروضها.

### من يوافق؟
- صاحب العرض + الوسيط (إن وجد)

### القيود:
- لا يستطيعان الرفض، فقط **موافقة** أو **اقتراح موعد جديد**
- **لا يريان** معلومات طالب الحجز (الاسم والهاتف)

## ✅ النقطة 9: الوسيط ولوحته + كيف يصير وسيطاً

### تسلسل الأدوار:
```
مستخدم عادي → يطلب وساطة ← توافق الإدارة → يصير وسيط
```

### 1. بيانات طلب الوساطة
- **فئة الوساطة:** شركة وساطة عقارية / مكتب عقاري / وسيط عقاري

- صورة البطاقة الشخصية (اختياري)
- **نوع الحساب:** مجاني / مدفوع

### 2. خطوات التفعيل
1. يرسل المستخدم طلب الوساطة
2. توافق الإدارة على الطلب
3. يقوم بتسديد الرسوم (إن وجدت — اختياري)
4. يتم **تفعيل حسابه كوسيط**
5. يُخرج من وضع "مستخدم عادي" ويجب تسجيل الدخول كوسيط

### 3. سرية المعلومات
- جميع بيانات الوسيط **سرية** ولا تظهر للجمهور.

## 💎 نظام إتمام المعاملة (Deal Completion System)

### 🟢 المرحلة 1: بطاقة الموعد التفاعلية (ما بعد المعاينة)

كل طرف يفتح واجهته:

```
🧑 صاحب العرض: [✅ أوافق] [❌ لا] [⏳ مهلة: 1-3 أيام]
🕵️ المشتري:     [✅ أنا جاد] [❌ مش معجب] [⏳ مهلة: 1-3 أيام]
🏢 الإدارة:     [📝 ملاحظات داخلية]
```

### 🟡 المرحلة 2: السيناريوهات والمهلة

- المهلة **تبدأ من وقت الضغط على الزر**
- يتم إشعار الأطراف بكل تحديث
- قبل انتهاء المهلة بـ **ربع المدة** ← إشعار للمستمهل "حسم قرارك"
- عند انتهاء المهلة ← "انتهت أولويتك"

| صاحب العرض | المشتري | النتيجة |
|-----------|---------|--------|
| ✅ + ✅ | — | صفقة مكتملة فوراً 🥳 |
| ✅ | ⏳ → ✅ | خلال المهلة → صفقة |
| ✅ | ⏳ → تجاهل | انتهت المهلة → تحذير |
| ⏳ → ✅ | ✅ | خلال المهلة → صفقة |
| ✅ | ❌ | يُقفل الموعد |
| تجاهل | تجاهل | الإدارة تتصل |

### 🟠 المرحلة 3: الإنهاء القسري (Admin Override)
- إذا تأكدت الإدارة من الصفقة ورفض أحد الطرفين الضغط → الإدارة **تغلق البطاقة قسراً**.
- تُحتسب الصفقة كاملة + العمولة.
- الطرف المتهرّب يواجه عقوبات.

### 🔴 المرحلة 4: تأكيد إتمام الصفقة

1. الطرفان يضغطان "قبول وإتمام المعاملة"
2. العرض يُحجز ← تُلغى باقي المواعيد ← إشعارات لأصحابها
3. **مندوب التطبيق يعبّي استمارة الإتمام**:

```
📋 استمارة إتمام المعاملة
═══════════════════════════════════════
بيانات البائع/المؤجر: (الاسم، الهوية، الهاتف)
بيانات المشتري/المستأجر: (الاسم، الهوية، الهاتف)
بيانات العرض: (تلقائي من النظام)
الثمن النهائي: (قد يختلف عن المعلن)
العمولة:
  - النسبة المتفق عليها: __%
  - القيمة النهائية: ___
  - ملاحظات الخصم: ___
المرفقات: [📎 عقد البيع/الإيجار] [📎 إثبات الدفع]
ملاحظات مندوب التطبيق:
متابعات مطلوبة من الشركة:
═══════════════════════════════════════
[✅ تأكيد الإتمام] [💾 حفظ كمسودة]
```

4. تُفعّل المكافآت (نقاط + خانات)
5. تقييم متبادل بين الطرفين

## 🧠 المطابقة الآلية (Auto-Matching)

### مبدأ عام
- المطابقة اليدوية **غير مطلوبة**.
- تعتمد **على السعر فقط** بهامش تسامح **±25%**.

### آلية العمل
1. عند إضافة عرض أو طلب → النظام يبحث عن مطابقات.
2. بعد النشر → إشعار للمستخدم + شاشة مطابقات.
3. المستخدم يفلتر النتائج.
4. بعد الاختيار → زر **"حجز موعد"**.

### ملاحظات
- تعمل تلقائياً عند إضافة/اعتماد طلب + نشر عرض جديد.
- النتيجة: إشعار للإدارة فقط.

---

## 💳 الدفع (Manual Settlement)
- لا تكامل API — تجنب مخاطر الإيقاف.
- المستخدم يختار باقة → تعليمات دفع من الإدارة (شام كاش/حوالة).
- يدفع → يرفع إثبات دفع → الإدارة تفعّل يدوياً.

## 📢 الإعلانات الخارجية

### الأماكن المقترحة
- بانر أعلى قائمة العروض (بعد شريط البحث).
- بانر بين كل 6-8 عرض في القائمة.

### المواصفات
- صورة بعرض الشاشة (Banner) + نص + زر "تواصل عبر واتساب".
- **ممنوع:** Pop-ups إجبارية · فيديو إجباري.

### المدة
- 7 أيام أو 30 يوم — تختارها الإدارة.

### الإدارة
- قسم مستقل "📢 الإعلانات" في لوحة التحكم → إضافة، حذف، تمديد، إيقاف.
- الإعلانات تحتاج **مراجعة قبل النشر**.
- المعلن يدفع (شام كاش/حوالة).

## 📌 ملاحظات إضافية

### منع التجاوز على العمولة
- التعهد الإلكتروني عند إضافة العرض/الطلب (ملزم قانونياً).
- الإدارة موجودة في المعاينة الميدانية.
- الإنهاء القسري للبطاقة عند التجاوز.
- الحظر النهائي + القائمة السوداء للمخالفين.
- الخصم من النقاط.

### نطاق السيارات
- كل ما ينطبق على العقارات (باستثناء حقول المساحة/الطابق/التشطيب) ينطبق على السيارات.
- المواعيد والمعاينة وإتمام المعاملة → نفس الآلية.

### قائمة مناطق السويداء
- جاهزة ومنقّحة في `docs/locations.json`.
- تحتاج مراجعة نهائية من صاحب المشروع.
