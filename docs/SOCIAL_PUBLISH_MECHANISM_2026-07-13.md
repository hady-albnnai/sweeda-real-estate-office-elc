# آلية النشر التلقائي على صفحات التواصل (فيسبوك + إنستغرام)

**التاريخ:** 2026-07-13  
**الحالة:** المرحلة 1 (جدولة + نقاط + إشارات) منفذة بالكامل

---

## الآلية الكاملة (خطوة بخطوة)

### 1. عند إضافة العرض (من المستخدم العادي)
**الملف:** `lib/screens/user/add_offer_screen.dart`

- يظهر **تشيك بوكس مفعّل تلقائياً** (يمكن إلغاؤه):
  > نشر العرض تلقائياً على صفحاتنا في فيسبوك وإنستغرام (وأي صفحات إضافية مضافة)

- عند الضغط على "نشر العرض للمراجعة الآن":
  - `i_soc = 1` (إذا مفعّل)
  - `socTxt` = **قالب منشور جاهز** يُولّد تلقائياً (دالة `_generateSocialPostText`)

**القالب الناتج (مثال):**
```
🏠 عرض جديد على عقارات السويداء

📍 شقة فاخرة في السويداء
عقار للبيع
💰 السعر: $120000
📌 المنطقة: ...

[الوصف الكامل]

للتفاصيل والحجز:
📱 تطبيق عقارات السويداء
#عقارات_السويداء #عروض_عقارية
```

### 2. عند مراجعة العرض (الإدارة)
**الملف:** `lib/screens/admin/offers_review_screen.dart`

- إذا `i_soc == 1` و `socTxt` موجود → تظهر **شارة زرقاء واضحة**:
  > 📣 سيُنشر تلقائياً على فيسبوك + إنستغرام (بعد الموافقة)

- عند الضغط على **"قبول"**:
  - يظهر تأكيد عادي ("سيتم نشر العرض...")
  - بعد النجاح تظهر رسالة:
    > ✅ تم نشر العرض • 📣 تم تفعيل النشر التلقائي على فيسبوك وإنستغرام

### 3. المنطق الحقيقي (القلب) — داخل Edge Function
**الملف:** `supabase/functions/admin-offers/index.ts`

عند `action === "review"` و `approve === true`:

```typescript
// بعد الموافقة الناجحة
if (iSoc === 1 && socTxt.length > 10) {
  // 1. علم أنه جاهز للنشر
  await supabaseAdmin.from('offers').update({ soc_pub: 1 }).eq('id', offerId);

  // 2. منح نقاط النشر الاجتماعي (pts.soc)
  await supabaseAdmin.rpc('award_points_safe', {
    p_user_uid: ownerUid,
    p_event_type: 'soc',
    p_points: 100,
  });

  // 3. تسجيل في سجل الإجراءات الإدارية
  await supabaseAdmin.rpc('log_admin_action', {
    p_action: 105,
    p_details: 'تم تفعيل النشر التلقائي على السوشيال (i_soc=1)',
  });
}
```

### 4. قيم حقل `soc_pub` (المعتمدة حالياً)

| soc_pub | المعنى                              | متى يحدث؟                  |
|---------|-------------------------------------|-----------------------------|
| 0       | لم يُفعّل النشر الاجتماعي          | التشيك بوكس غير مفعّل      |
| **1**   | **جاهز للنشر** (تمت الجدولة)       | بعد الموافقة + i_soc=1     |
| 2       | نُشر فعلياً على السوشيال (حقيقي)   | المرحلة 2 (لاحقاً)        |

## المرحلة 2 (النشر الحقيقي) — لاحقاً

نحتاج:
- Facebook Page Access Token (مع صلاحيات `pages_manage_posts`)
- Instagram Business Account ID
- Meta App معتمدة

ثم سنضيف:
- Edge Function جديدة: `publish-to-social`
- استدعاء Graph API:
  ```ts
  POST https://graph.facebook.com/v19.0/{page_id}/feed
  {
    message: socTxt,
    link: "https://app.sweeda-realestate.com/offer/xxx"
  }
  ```
- بعد نجاح النشر: `soc_pub = 2`

## ملاحظات هامة

- **النشر لا يحدث فوراً** عند إضافة العرض (يجب موافقة الإدارة أولاً).
- المستخدم يستطيع إلغاء التشيك بوكس في أي وقت قبل الإرسال.
- كل العروض (حتى المضافة من الإدارة) تحترم `i_soc`.
- لا يؤثر على طلبات الفيديو أو التحقق من الهاتف.

**هل تريد الآن:**
1. اختبار التدفق كامل (من إضافة → موافقة → soc_pub=1)؟
2. إضافة زر "نشر يدوي الآن" في شاشة المراجعة؟
3. الانتقال إلى المرحلة 2 (أعطني التوكنات)؟

جاهز لأي خطوة تالية.
